# Stitch 화면 제작 프롬프트

아래 내용을 Stitch에 그대로 입력한다.

---

고령층 손님이 일자리를 선택하고 공용 번호표를 발급받는 모바일 앱과, 담당자가
일자리와 접수 세션을 관리하는 관리자 모바일 앱의 화면을 디자인해줘.

## 서비스 배경

- 주요 사용자는 스마트폰 사용이 익숙하지 않은 60~80대다.
- 손님은 사업단을 탐색 기준으로 사용할 수 있지만, 반드시 **정확한 일자리
  하나를 선택한 뒤** 번호표를 발급받아야 한다.
- 번호는 일자리별 번호가 아니라 모든 일자리가 공유하는 공용 순번이다.
- 번호표 발급에는 선택한 `jobId`와 열린 `sessionUid`가 모두 필요하다.
- 전화번호는 대한민국 국가 코드가 포함된 형식으로 입력받는다.
- 이미 발급받은 손님이 다시 요청하면 새 번호가 아니라 기존 번호를 보여준다.
- 관리자는 일자리를 만들고, 해당 일자리의 접수 세션을 열고 닫는다.

## 공통 디자인 원칙

- 한국어 모바일 화면, 세로형 390×844 기준으로 제작한다.
- 밝고 안정적인 공공기관 서비스 분위기로 표현한다.
- 배경은 따뜻한 흰색, 주요 색상은 진한 남색 또는 짙은 청록색을 사용한다.
- 본문 글자는 최소 18px, 주요 제목 26~32px, 핵심 번호는 64px 이상으로 한다.
- 터치 영역은 최소 48px, 주요 버튼 높이는 56~64px로 한다.
- 텍스트와 배경의 명암비는 WCAG AA 이상을 유지한다.
- 아이콘만 사용하지 말고 항상 명확한 한국어 라벨을 함께 표시한다.
- 한 화면에서 하나의 핵심 행동만 강조한다.
- 긴 설명, 작은 보조 링크, 장식적인 애니메이션, 복잡한 표와 그래프는 배제한다.
- 뒤로 가기, 취소, 오류 복구 방법을 항상 명확하게 제공한다.
- 전화번호 전체, 인증 토큰, 내부 DB ID는 화면에 노출하지 않는다.
- 공용 번호와 선택한 일자리명을 항상 함께 표시해 번호의 의미를 혼동하지 않게
  한다.

# 1. 손님용 모바일 앱

다음 화면을 하나의 완성된 사용자 흐름으로 제작한다.

## 화면 1: 시작 화면

- 상단 제목: `일자리 접수 번호표`
- 설명: `원하는 일자리를 선택하고 번호표를 받아주세요.`
- 큰 기본 버튼: `일자리 찾기`
- 하단에는 운영기관명과 `도움이 필요하신가요? 직원에게 문의해주세요.` 안내만
  간결하게 표시한다.

## 화면 2: 일자리 목록

- 제목: `접수할 일자리를 선택해주세요`
- 사업단 필터는 `전체`, `중부사업단`, `남부사업단`, `북부사업단` 형태의 큰
  선택 칩으로 제공한다.
- 필터는 탐색 용도일 뿐이며 사업단 선택만으로 다음 단계에 갈 수 없게 한다.
- 각 일자리 카드는 다음 내용을 표시한다.
  - 일자리명
  - 사업단명
  - 접수 상태 배지: `접수 중` 또는 `접수 마감`
  - 큰 버튼: `이 일자리 선택`
- 열린 접수 세션이 없는 일자리는 선택 버튼을 비활성화하고 `현재 접수할 수
  없습니다`라고 표시한다.
- 검색 결과가 없을 때 빈 화면 안내와 `전체 일자리 보기` 버튼을 제공한다.

## 화면 3: 일자리 확인

- 제목: `이 일자리로 접수할까요?`
- 선택한 일자리명과 사업단명을 큰 확인 카드로 표시한다.
- 안내: `번호표는 모든 일자리가 함께 사용하는 공용 순번입니다.`
- 기본 버튼: `전화번호 입력하기`
- 보조 버튼: `다른 일자리 선택`
- 내부 `jobId`, `sessionUid`는 화면에 표시하지 않는다.

## 화면 4: 전화번호 입력

- 제목: `전화번호를 입력해주세요`
- 설명: `이미 번호표를 받으셨다면 같은 번호를 다시 보여드립니다.`
- 대한민국 번호 입력에 적합한 큰 숫자 키패드를 제공한다.
- 입력값은 읽기 쉬운 형태로 구분해 보여주되 서버 전송 시 E.164 형식을 사용한다.
- 입력값 아래에 예시 `010-1234-5678`을 표시한다.
- 기본 버튼: `번호표 받기`
- 전화번호가 완성되기 전에는 버튼을 비활성화한다.
- 개인정보 안내는 한 문장으로 제한한다:
  `전화번호 원문은 저장하지 않으며 중복 발급 확인에만 사용합니다.`

## 화면 5: 발급 처리 중

- 제목: `번호표를 발급하고 있습니다`
- 단순한 진행 표시와 `잠시만 기다려주세요.` 문구를 제공한다.
- 중복 탭을 막고 뒤로 가기 시 경고한다.
- 10초 이상 지연되면 `다시 시도` 버튼을 표시한다.

## 화면 6: 번호표 발급 완료

- 가장 중요한 공용 번호를 화면 중앙에 64px 이상으로 표시한다.
- 번호 위 라벨: `공용 번호`
- 번호 아래에 선택한 일자리명과 사업단명을 표시한다.
- 신규 발급이면 `번호표가 발급되었습니다.`를 표시한다.
- 기존 번호 재조회이면 `이미 받은 번호표입니다.`와 `새 번호는 발급되지
  않았습니다.`를 표시한다.
- 발급 시각을 읽기 쉬운 한국어 형식으로 제공한다.
- 기본 버튼: `확인`
- 보조 버튼: `직원에게 보여주기` 모드로 전환하며 번호와 일자리명을 더 크게
  표시한다.

## 화면 7: 오류 및 상태 화면

동일한 디자인 체계로 다음 상태를 각각 제작한다.

- `접수가 마감되었습니다` — 다른 일자리 찾기 버튼
- `일자리 정보를 찾을 수 없습니다` — 목록으로 돌아가기 버튼
- `전화번호를 확인해주세요` — 입력 화면으로 돌아가기 버튼
- `인터넷 연결을 확인해주세요` — 다시 시도 버튼
- `번호표를 발급할 수 없습니다` — 다시 시도와 직원 문의 안내

오류 코드나 서버 내부 메시지는 노출하지 않는다.

# 2. 관리자 모바일 앱

손님용 앱과 같은 디자인 언어를 사용하되 운영 상태를 더 명확하게 표현한다.

## 화면 1: 관리자 인증

- 제목: `관리자 로그인`
- 현재 bootstrap 운영을 고려해 관리자 접근 토큰 입력 화면으로 표현하되,
  실제 공개 운영에서는 계정 로그인으로 교체될 수 있는 구조로 만든다.
- 토큰은 마스킹하고 저장 여부를 명확하게 표시한다.
- 기본 버튼: `로그인`
- 인증 실패 시 `관리자 정보를 확인해주세요.`만 표시한다.

## 화면 2: 운영 현황

- 제목: `접수 운영 현황`
- 상단에 현재 공용 마지막 번호를 표시하지 않는다. API가 제공하지 않는 값을
  임의로 만들지 않는다.
- 일자리 카드는 일자리명, 사업단, 접수 세션 상태를 표시한다.
- 상태는 `접수 중`, `접수 마감`, `세션 없음`으로 구분한다.
- 주요 버튼: `새 일자리 만들기`
- 각 카드에는 상황에 따라 `접수 시작`, `접수 마감` 버튼을 제공한다.

## 화면 3: 일자리 생성

- 제목: `새 일자리 만들기`
- 입력 항목: `일자리명`, `사업단명`
- 입력 내용을 확인하는 요약 영역을 제공한다.
- 기본 버튼: `일자리 만들기`
- 성공 후 해당 일자리 상세 화면으로 이동한다.

## 화면 4: 일자리 상세 및 접수 시작

- 일자리명과 사업단명을 크게 표시한다.
- 열린 세션이 없으면 기본 버튼 `접수 시작`을 제공한다.
- 접수 시작 전 확인 팝업:
  `이 일자리의 번호표 접수를 시작합니다.`
- 접수 중이면 상태와 세션 생성 시각을 표시하고 `접수 마감` 버튼을 제공한다.
- 내부 UUID는 일반 화면에 표시하지 않는다.

## 화면 5: 접수 마감 확인

- 경고 제목: `접수를 마감할까요?`
- 설명: `마감 후 새로운 번호표는 발급할 수 없습니다. 이미 번호표를 받은
  손님은 기존 번호를 다시 확인할 수 있습니다.`
- 위험 버튼: `접수 마감`
- 취소 버튼: `계속 접수하기`
- 위험 버튼은 붉은색을 사용하되 다른 기본 행동과 명확히 구분한다.

## 화면 6: 접수 이력

- 날짜와 일자리 기준 필터를 제공한다.
- 각 이력은 공용 번호, 일자리명, 사업단, 발급 시각만 표시한다.
- 전화번호 원문, HMAC, 인증정보는 노출하지 않는다.
- API가 제공하지 않는 개인정보나 통계는 임의로 추가하지 않는다.

# 3. 프로토타입 연결

다음 순서로 클릭 가능한 프로토타입을 연결한다.

손님용:

`시작 → 일자리 목록 → 일자리 확인 → 전화번호 입력 → 처리 중 → 발급 완료`

관리자용:

`로그인 → 운영 현황 → 일자리 생성 또는 상세 → 접수 시작 → 접수 마감`

모든 주요 화면에 정상 상태뿐 아니라 loading, empty, disabled, validation error,
server error 상태를 함께 제작한다. 최종 결과에는 손님용 핵심 화면과 관리자용
핵심 화면을 각각 별도의 모바일 화면 세트로 보여주고, 재사용 가능한 색상,
타이포그래피, 버튼, 카드, 입력창, 상태 배지 구성요소를 디자인 시스템으로
정리해줘.
